# Steam 假入库 / 解锁工具类型分类

> 这个文档用于后续分析新软件、新脚本时统一分类。  
> 一个工具可能同时属于多个分类：例如「irm 脚本入口」+「DLL 代理注入」+「Defender 排除」。

## 一、按入口 / 分发方式分类

### 1. 终端 irm / iex / PowerShell 一行命令

典型形式：

```powershell
irm xxx | iex
iwr xxx -OutFile get.ps1; powershell -File get.ps1
```

常见行为：

- 下载远程 `.ps1` 到 `%TEMP%`、`%USERPROFILE%` 或当前目录。
- 再由下载脚本继续下载 DLL / EXE / VDF。
- 可能自我提权。
- 可能添加 Defender 排除项。
- 可能修改 SmartScreen / CI Policy。
- 可能写入 `HKCU:\Software\Valve\Steamtools`。

当前已见例子：

- `scripts/steam-run.com/*`
- `scripts/steam.run/*`
- `scripts/cdk.ruku.run/cdk.ruku.run.ps1`
- `scripts/cdk.ruku.run/a.ps1`

重点清理项：

- `%USERPROFILE%\get.ps1`
- `%TEMP%\*.ps1`
- `${ScriptRoot}\a.ps1`
- 下载落地的 DLL / EXE
- Defender 排除项
- Steamtools 注册表
- SmartScreen / CI Policy 注册表

---

### 2. 一键入库软件 / GUI 软件 / EXE 启动器

典型形式：

- 用户下载一个 `.exe`
- 双击运行
- 软件自己下载资源、注入 Steam、管理 AppID / 清单

常见行为：

- 在 `%APPDATA%`、`%LOCALAPPDATA%`、`%ProgramData%` 创建软件目录。
- 在 Steam 目录写入 DLL / 配置。
- 可能创建启动项、计划任务、服务。
- 可能常驻后台进程。
- 可能添加 Defender 排除项。

当前已见相关痕迹：

- `steamcdk.exe`
- `steam_auth`
- `SteamAuthDaemon`

后续分析重点：

- 软件安装目录
- 程序进程名
- 启动项 / 计划任务 / 服务
- 自更新目录
- 下载缓存目录
- 卸载后残留目录

---

### 3. 开源 Hook / 内核替换类工具

典型形式：

- 用户把 DLL 放进 Steam 根目录。
- 通过 DLL 代理 / hook 替换 Steam 加载链。
- 通过配置文件或 Lua 脚本定义解锁逻辑。

常见文件：

- `${SteamPath}\dwmapi.dll`
- `${SteamPath}\xinput1_4.dll`
- `${SteamPath}\hid.dll`
- `${SteamPath}\version.dll`
- `${SteamPath}\winmm.dll`
- `${SteamPath}\OpenSteamTool.dll`
- `${SteamPath}\opensteamtool.toml`
- `${SteamPath}\config\lua`

当前已见例子：

- [OpenSteam001/OpenSteamTool](https://github.com/OpenSteam001/OpenSteamTool)
- 早期脚本下载的 `dwmapi.dll`、`xinput1_4.dll`、`hid.dll`、`zlib1.dll`

重点清理项：

- Steam 根目录 DLL 代理文件
- 工具主 DLL
- 工具配置文件
- Lua / plugin 配置目录

注意：

- 不应直接删除整个 Steam 目录。
- `config\lua` 可能有用户自写脚本，应设为 `medium` + `confirm: true`。

---

### 4. SteamTools / GreenLuma / Koalageddon / CreamAPI 类工具

这类通常不是简单脚本，而是成熟工具或框架。

常见影响：

- Steam 根目录代理 DLL
- Steam `steamapps` / `depotcache` 清单文件
- App manifest 修改
- App ticket / entitlement 相关数据
- `HKCU:\Software\Valve\Steamtools`
- 工具自身配置目录

可能出现的文件 / 关键词：

- `GreenLuma*.dll`
- `SteamTools*.exe`
- `SteamTools*.dll`
- `Koalageddon*.dll`
- `cream_api.dll`
- `cream_api.ini`
- `SmokeAPI*.dll`
- `appmanifest_*.acf`
- `*.manifest`
- `*.lua`

清理策略：

- 明确落地文件：可自动清理。
- App manifest / depotcache：建议先列出，默认确认后清理。
- AppTicket / ETicket：需要精确 AppID，不建议泛删。

---

### 5. 配置 / Manifest / Depot 注入类

这类不一定有 EXE 或 DLL，主要通过修改 Steam 配置和清单达到效果。

常见位置：

- `${SteamPath}\config\appdata.vdf`
- `${SteamPath}\appcache\appdata.vdf`
- `${SteamPath}\appcache\packageinfo.vdf`
- `${SteamPath}\package\beta`
- `${SteamPath}\steamapps\appmanifest_*.acf`
- `${SteamPath}\depotcache\*.manifest`
- `${SteamPath}\depotcache\*.lua`
- `%LOCALAPPDATA%\Steam\localData.vdf`

清理策略：

- 明确由假入库脚本生成的文件：可加入 `targets.json`。
- Steam 正常清单文件：只能作为可疑项列出，不应默认删除。
- 如果要删除 `appmanifest_*.acf`，必须有明确 AppID / 来源证据。

---

### 6. 系统策略 / 安全软件绕过类

这类是高风险行为，不一定直接和 Steam 文件相关。

常见行为：

- 添加 Defender 排除项：
  - `${SteamPath}`
  - `%APPDATA%\Stool`
  - `.exe`
  - `.dll`
- 关闭 SmartScreen：
  - `SmartScreenEnabled=Off`
  - `EnableSmartScreen=0`
- 修改 CI Policy：
  - `VerifiedAndReputablePolicyState=0`
- 暂停 Windows Update：
  - `HKLM:\SOFTWARE\Microsoft\WindowsUpdate\UX\Settings`

清理策略：

- 统一标记为 `high`。
- 必须 `confirm: true`。
- 清理前必须完整列出。

## 二、按影响范围分类

| 影响范围 | 例子 | 默认风险 | 清理方式 |
|---|---|---:|---|
| Steam 根目录文件 | `dwmapi.dll`, `xinput1_4.dll`, `OpenSteamTool.dll` | low/medium | 可清理 |
| Steam 配置/缓存 | `steam.cfg`, `appdata.vdf`, `packageinfo.vdf` | low/medium | 可清理或确认 |
| AppData 工具目录 | `%APPDATA%\Stool` | low | 可清理 |
| LocalAppData 工具目录 | `%LOCALAPPDATA%\steam`, `%LOCALAPPDATA%\Steam\localData.vdf` | medium | 确认清理 |
| 注册表工具项 | `HKCU:\Software\Valve\Steamtools` | low/medium | 可清理 |
| 启动项/计划任务/服务 | `SteamAuthDaemon` | medium/high | 确认清理 |
| Defender 排除项 | Steam 路径、DLL/EXE 扩展名 | high | 确认清理 |
| SmartScreen / CI Policy | 系统安全策略 | high | 确认清理 |
| Steam Apps/AppTicket | `HKCU:\Software\Valve\Steam\Apps\<appid>` | high | 仅精确 AppID 清理 |
| Windows Update 策略 | 暂停更新到未来年份 | high | 确认清理 |

## 三、当前已分析对象归类

| 对象 | 入口类型 | 技术类型 | 主要清理项 |
|---|---|---|---|
| `steam-run.com` 系列 | irm / PowerShell 脚本 | DLL 代理 + Steamtools 注册表 | Steam 根目录 DLL、`%APPDATA%\Stool`、Steamtools、Defender |
| `steam.run` / `cdks.run` | irm / PowerShell 脚本 | 单 DLL 替换 + Steamtools 注册表 | `xinput1_4.dll`、`dwmapi.dll`、`%LOCALAPPDATA%\steam`、Steamtools |
| `steam.work` / `steam.icu` | PowerShell 脚本 | DLL 代理 + localData | `xinput1_4.dll`、`localData.vdf`、Defender |
| `steamcdkey.cn` | 大型 PowerShell 脚本 | DLL 代理 + Manifest + 系统策略 | DLL、Steamtools、CI Policy、Windows Update、Defender |
| `cdk.ruku.run/a.ps1` | 下载器 + PowerShell 脚本 | EXE 落地 + 系统策略 | `steamcdk.exe`、SmartScreen、CI Policy、SteamAuthDaemon |
| OpenSteamTool | 开源 Hook / 内核替换 | DLL 代理 + Lua 配置 | `OpenSteamTool.dll`、`dwmapi.dll`、`xinput1_4.dll`、`opensteamtool.toml`、`config\lua` |

## 五、加入 `targets.json` 的原则

1. **能精确定位的才加入**
   - 例如 `${SteamPath}\OpenSteamTool.dll` 可以加入。
   - 例如整个 `HKCU:\Software\Valve\Steam\Apps` 不应加入。

2. **能确认是工具残留的优先加入**
   - Steam 正常文件不要因为名字像就删除。

3. **涉及用户数据 / Steam 正常配置必须确认**
   - `config\lua`
   - `localData.vdf`
   - `steamapps\appmanifest_*.acf`

4. **系统安全策略一律高风险**
   - Defender
   - SmartScreen
   - CI Policy
   - Windows Update

5. **一个软件可以拆成多个规则**
   - 程序文件：low
   - 配置目录：medium + confirm
   - 系统策略：high + confirm
