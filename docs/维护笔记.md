# SteamSPA 本地维护笔记

> 本文档仅作为本地维护记录，`docs/` 不上传 GitHub。  
> 对外公开仓库保持极简：`.gitignore`、`README.md`、`uninstall.ps1`。

## 当前项目定位

SteamSPA 只做一件事：

1. 识别 Steam 假入库 / 解锁脚本留下的残留。
2. 展示扫描结果。
3. 用户确认后清理。

对外只维护一个脚本：

```text
uninstall.ps1
```

不再维护：

- `targets.json`
- `targets.schema.json`
- 远程 bootstrap 脚本
- 快捷方式生成器
- 对外 docs / dev / scripts 目录

## 对外使用方式

推荐：

```powershell
irm https://raw.githubusercontent.com/ZERONE2077/SteamSPA/refs/heads/main/uninstall.ps1 | iex
```

CDN：

```powershell
irm https://cdn.jsdelivr.net/gh/ZERONE2077/SteamSPA@main/uninstall.ps1 | iex
```

流程固定为：

```text
扫描 → 显示残留 → 输入 Y 才清理 → 其他输入取消
```

## 当前脚本结构

`uninstall.ps1` 是自包含脚本：

- 规则内置在 `$EmbeddedTargetsJsonBase64`
- 运行时按 UTF-8 解码为 JSON
- 源码保持 ASCII / 无 BOM，避免 `irm | iex` 和 Windows PowerShell 5.1 编码问题
- 中文提示通过 Base64 解码输出
- 报告写入 `temp/logs/`
- 文件备份写入 `temp/backups/`

保留的可选参数：

- `-Risk low,medium,high`
- `-Only <规则ID>`
- `-NoBackup`
- `-NoPause`

普通用户不需要参数。默认会展示 low / medium / high 全部风险项，但仍需确认后才清理。

## 已归纳的假入库 / 解锁类型

### 1. PowerShell 一行命令 / irm 入口

典型形式：

```powershell
irm xxx | iex
iwr xxx -OutFile get.ps1; powershell -File get.ps1
```

常见残留：

- `%USERPROFILE%\get.ps1`
- `%TEMP%\*.ps1`
- `${ScriptRoot}\a.ps1`
- Steam 根目录 DLL
- `HKCU:\Software\Valve\Steamtools`
- Defender 排除项

已见来源：

- `steam-run.com`
- `steam.run`
- `cdks.run`
- `cdk.ruku.run`
- `steam.work` / `steam.icu`

### 2. DLL 代理 / Hook 类

常见文件：

- `${SteamPath}\dwmapi.dll`
- `${SteamPath}\xinput1_4.dll`
- `${SteamPath}\hid.dll`
- `${SteamPath}\zlib1.dll`
- `${SteamPath}\version.dll`
- `${SteamPath}\user32.dll`
- `${SteamPath}\wtsapi32.dll`
- `${SteamPath}\OpenSteamTool.dll`
- `${SteamPath}\opensteamtool.toml`
- `${SteamPath}\config\lua`

策略：

- 明确工具落地文件可以清理。
- 可能包含用户自定义内容的目录标记为 `medium`，确认后清理。
- 不删除整个 Steam 目录。

### 3. Steamtools / 注册表类

常见注册表：

```text
HKCU:\Software\Valve\Steamtools
```

常见值：

- `packageinfo`
- `steamclient`
- `s`
- `c`
- `iscdkey`
- `ActivateUnlockMode`
- `AlwaysStayUnlocked`
- `notUnlockDepot`

策略：

- 精确匹配工具注册表项。
- 不泛删 Steam 正常注册表。

### 4. 配置 / Manifest / Depot 注入类

常见位置：

- `${SteamPath}\config\appdata.vdf`
- `${SteamPath}\appcache\appdata.vdf`
- `${SteamPath}\appcache\packageinfo.vdf`
- `${SteamPath}\package\beta`
- `%LOCALAPPDATA%\Steam\localData.vdf`

策略：

- 明确由假入库脚本生成的文件可以加入规则。
- `appmanifest_*.acf`、`depotcache`、Ticket 类内容不泛删，必须有明确 AppID / 来源证据。

### 5. 安全策略 / Defender 绕过类

常见行为：

- 添加 Defender 排除路径：`${SteamPath}`、`%APPDATA%\Stool`
- 添加排除扩展名：`exe`、`dll`、`.exe`、`.dll`
- 修改 SmartScreen
- 修改 CI Policy
- 暂停 Windows Update

策略：

- 统一标记为 `high`
- 必须清楚展示给用户
- 用户确认后才清理

### 6. GUI / 安装包类

已见例子：

- SteamKing

常见残留：

- 安装目录
- `%LOCALAPPDATA%` 应用目录
- 卸载注册表
- 常驻进程
- 启动项 / 服务 / 计划任务

策略：

- 程序文件和本体目录可作为 `medium`。
- 系统级服务、任务、策略按 `medium/high` 处理。

## 新增识别项维护流程

新增一个软件 / 脚本时，按下面流程处理：

1. 保存原始样本到本地 `scripts/<来源>/`。
2. 只做静态分析，不直接执行陌生脚本。
3. 总结它写入 / 删除 / 修改的内容：
   - 文件
   - 注册表
   - Defender 排除项
   - SmartScreen / CI Policy
   - 计划任务 / 服务 / 启动项
   - 进程
   - Steam 配置 / Manifest / Ticket
4. 判断风险等级：
   - `low`：明确工具残留，误删风险低
   - `medium`：可能涉及用户配置或正常软件目录
   - `high`：系统安全策略、Defender、SmartScreen、CI Policy、Ticket / AppID 类
5. 把确认后的规则维护进 `uninstall.ps1` 内置规则。
6. 本地测试：
   - 能解析脚本
   - 能扫描
   - 取消清理不会删除
   - 输入 `Y` 时清理逻辑符合预期
7. 提交推送。

## ruku.run 补充复查

`scripts/cdk.ruku.run/cdk.ruku.run.ps1` 是入口脚本，会下载 `%TEMP%\1.ps1` 并执行；实际落地行为在 `scripts/cdk.ruku.run/a.ps1` 中。

已确认 ruku 相关残留：

- `${SteamPath}\steamcdk.exe`：由 `a.ps1` 下载到 Steam 根目录并启动。
- `HKLM:\SYSTEM\CurrentControlSet\Control\CI\Policy\VerifiedAndReputablePolicyState`
- SmartScreen 相关注册表：`HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer`、`HKLM:\SOFTWARE\Policies\Microsoft\Windows\System`、`HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\SmartScreen`
- `HKCU:\Software\Microsoft\Windows\CurrentVersion\Run\SteamAuthDaemon`
- `steam_auth` 进程
- Defender 排除项：Steam 路径、`.dll`、`.exe`
- `%TEMP%\1.ps1`

处理结论：

- `downloaded-steamcdk-exe` 规则已覆盖 `${SteamPath}\steamcdk.exe`。
- 默认风险范围已改为 `low,medium,high`，因此 high 风险项会默认展示，但仍需用户确认才清理。
- ruku 相关规则 sources 同时记录入口 `cdk.ruku.run.ps1` 和实际脚本 `a.ps1`。
